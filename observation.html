Observations & RESULTS

1. Mathematical Foundation

A Linear Feedback Shift Register (LFSR) is a shift register whose input bit is a linear function of its previous state. The most common linear function of single bits is XOR. The operation is governed by a characteristic polynomial.

$P(x) = x^n + c_{n-1}x^{n-1} + ... + c_1x + 1$

Where $n$ is the number of stages (bits) and $c_i$ represents the tap coefficients (either 0 or 1).

2. Maximal Length Sequences (m-sequences)

It is observed that the LFSR cycles through a sequence of states. The maximum number of unique states an $n$-bit LFSR can have is:

$L = 2^n - 1$

This is called an m-sequence. The "all-zero" state is excluded because the XOR sum of 0s is 0, which would result in the register remaining in that state forever.

3. Observations on Tap Selection

  • Primitive Polynomials: If the feedback taps are chosen based on a primitive polynomial, the LFSR will achieve the maximum period.
  • Non-Primitive Taps: If the taps do not form a primitive polynomial, the sequence will repeat much sooner, resulting in multiple smaller cycles.

4. Properties of the Generated Sequence

During the lab experiment, we observe the following properties of m-sequences:

  • Balance Property: The number of 1s is always $(L+1)/2$ and the number of 0s is $(L-1)/2$.
  • Run Property: Short runs of 1s or 0s occur more frequently than long runs.
  • Autocorrelation: The sequence has a sharp autocorrelation peak, making it ideal for synchronization in CDMA systems.
